Qiwi 说说No.01
设计田野手记

一份给个人博客的设计语言。克制装饰，把表达让给文字、留白、与一抹批注的色。背景是一张微微泛黄的纸，主色是一缕焦糖墨；七彩留给标签和提示，不参与正文阅读。

PLATE I · 2026 像翻一本插画随笔。图片是图片，文字是文字，并置而非叠加。

不做遮罩。文字永远在图片外，靠白色小卡片承托，避免"深浅都看不清"的困境。

16:7 主图 + 4:3 双联。一大两小，模仿杂志跨页 plate 的版式节奏。

§01 Principles

设计是把不必要的东西删掉

六条信念，决定 Qiwi 在每一处的取舍。不是规则，是性格。

i

平铺胜于堆叠

分组靠间距和细线，不靠卡片与阴影。一张纸足够撑起任何信息密度。

no shadows · no nested cards · only rules & rhythm
ii

纸感优先于屏幕感

暖白底、墨灰字、克制的边线。让屏幕模仿一本印刷品该有的气息。

paper #fafaf9 · ink #3f3f46 · all sans, well-spaced
iii

颜色是批注，不是装潢

七彩光谱不参与正文 —— 它们只在标签、引用、提示这些"边页注脚"出现。

hue tokens · semantic only · never as full background
iv

俏皮藏在细节里

正文沉静，但彩色细条、对话式微文案、hover 的横向位移，会在不经意处闪一下。

3px gradient stripe · translateX hover · friendly micro-copy
v

中英文是一对孪生兄妹

每个 token 同时为汉字和拉丁字母准备，字怀和 x-高度都校准过。汉字永不斜体。

paired stacks · CJK never italic · Latin italic only after CJK
vi

能少一根线就少一根

分隔线 1px，圆角不过 12px，阴影几乎不存在。视觉重量都让给字。

1px hairlines · radius ≤ 12px · shadow ≈ none
§02 Typography

两套字，各司其职

无衬线说话，等宽报数据。汉字与拉丁字母配对挑选过 —— 字怀、x-高度、灰度都尽量贴。

§03 Color

七色批注，一缕焦糖

一首小调式的色板：红、橙、黄、绿、青、蓝、紫，外加一缕焦糖墨。点击任何 token 复制变量名。

主色 / Brand

焦糖墨 Caramel
#8b7355 · --caramel
奶咖 Latte

七色面板从博客的语义提示沿用而来。文字色 + 16% 透明背景一对出现，在浅深模式各自校准对比度。

§04 Spacing & Layout

八的倍数，留白为先

所有间距都是 8px 的倍数（小处补 4）。当犹豫"该用多大"时，往往是上一档而非下一档 —— 留白几乎不会出错。

最大宽 1600px。两侧 spacer 让主内容在大屏不至于贴边，侧栏永远在右。即刻条出现时，跨 main + aside 两列。

留白哲学 / Whitespace

章节之间留 --s-9 (96px) 以上；段落之间 --s-4 (16px)。文章正文最大 960px，文段 64ch —— 让眼睛不必横扫整页。

§05 Shape · Radius · Depth

圆角克制，深度靠线条

圆角不是装饰 —— 它在按钮、徽章、输入框、文章头图这类"可触摸/视觉块"上出现。分隔线、薄边一律方角。

圆角阶 / Radius

--r-0 · 0
--r-sm · 4
--r-md · 8
--r-lg · 12
--r-pill

深度 / Depth

Qiwi 不用阴影制造层级 —— 用 线条与背景微差。三个层级，足以应付所有场景。

Z0 · 纸面

正文所在的层。无任何边界，只靠间距。

Z1 · 微凹

代码块、引用、被嵌入的次级内容。底色比纸面深一阶。

Z2 · 框线

需要明确边界的容器：表单、导航条、组件展位。

§06 Components

组件是段落，不是积木

每个组件先想"它是文中的什么角色" —— 标题？引语？脚注？想清楚了，样式自然就出来了。

§07 Iconography

图标是笔画，不是图画

线性、1.5px stroke、20×20 容器，圆头。能用文字说清楚的事情，不用图标。

样例 / Samples

推荐外部库：Lucide 或 Phosphor (light) —— 都默认是 1.5px / 20px，与本规范无缝对接。

§08 Motion

动效是呼吸，不是表演

三档时长，一种缓动 —— cubic-bezier(.25, .1, .25, 1)。任何超过 420ms 的动画都需要专门的理由。

时长 · Duration

--dur-fast
160 ms
--dur-base
240 ms
--transition
300 ms
--dur-slow
420 ms

点击任意条让小球跑一遍。

用法 · When to animate

  • fast 160ms 颜色 / hover / focus 状态
  • base 240ms / 300ms 主题切换、淡入淡出、文章列表滑动
  • slow 420ms 大块面切换、页面间转场（上限）
§09 Voice & Tone

说话像朋友，不像产品经理

写下来的东西首先得是"读得下去的"。第一人称、短句、偶尔自嘲，避免营销腔与缩写黑话。

● 这样说

"这是测试站点，看到怪怪的就当看到怪怪的。"

"颜色测试：红提醒、橙提示、黄重点……"

"今天写一点关于 typography 的笔记。"

温度感、第一人称、能讲清楚就讲清楚。

● 不这样说

"欢迎您访问本站，本站致力于提供优质阅读体验。"

"赋能用户全链路认知系统升级。"

"Click here to learn more about our exciting features!"

官腔、行业黑话、过度热情的英语营销腔。

§10 Do & Don't

能不做的事，就别做

几条最常踩的雷。每一条都对应一条原则的反面。

DO · 这样做
  • 用细线 + 留白分组信息
  • 正文限制在 64ch / 文章 960px 以内
  • 颜色作为脚注，不作为底色铺面
  • 先想"这是文中的什么角色"
  • 中英文之间留半角空格
  • Hover 用横向位移，不用阴影
DON'T · 别做
  • 用阴影或大圆角制造卡片层级
  • 让段落横跨整屏宽度
  • 把汉字设成斜体 italic
  • 把斜体英文塞在两个汉字之间
  • 用饱和色块铺满区域
  • 把所有按钮都做成主色 —— 主色因此失重
§11 CJK · 中英文混排细则

汉字不斜体，空格要补全

中英文混排是 Qiwi 写作的常态。这一节把容易踩的细节列在一起。

CJK · italic

汉字绝不使用斜体。方块字不为斜向设计而生 —— 强行倾斜会破坏字形结构，看起来像渲染错误。

需要强调时用：font-weight 600、不同颜色、或下划线。

这是一段中文 这是一段中文
EN · italic

英文 italic 可以用，但不要把斜体英文夹在两个汉字之间 —— italic 字怀向右下倾，会和左侧的方块字"撞肩"。

允许的位置：句首、句尾，或被英文/标点包围的中段。

这是一段 middle 文字 这是一段文字 middle word
spacing

中英文之间留半角空格 —— 正文里手动加，标题里也别省。

这是绝大多数中文排版指南的共识。也使行内代码与汉字之间不挤压。

使用 typography设计 使用 typography 设计
punctuation

中文句中用全角逗号和句号，英文句子里用半角。引号同理 —— "中文引号"配汉字，"English quotes" 配英文。

"今天写一点 typography 笔记。" — Qiwi
numbers

数字与单位之间留半角空格："4 分钟阅读" 而不是 "4分钟阅读"；除非是日期紧凑写法 "2026-04-29"。

17 px / 1.7 行高 · 4 分钟阅读
monospace

等宽体（mono）只用在真的代码 / token 名 / 时间戳，不用作排版花哨手段。所有等宽都内嵌 code 或带 mono 类。

--font-sans 是默认字体栈
